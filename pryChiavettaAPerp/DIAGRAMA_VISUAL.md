## ?? Protección contra Inyección SQL

```
CASO 1: Entrada maliciosa
?????????????????????????
Entrada: txtNombre = "'; DROP TABLE Usuario; --"

? MÉTODO INSEGURO (Concatenación):
   SQL = "INSERT INTO Usuario (Nombre) VALUES ('" + txtNombre + "')"
   SQL = "INSERT INTO Usuario (Nombre) VALUES (''; DROP TABLE Usuario; --')"
   RESULTADO: ˇSe ejecutaría DROP TABLE! ??

? MÉTODO SEGURO (Parámetros):
   SQL = "INSERT INTO Usuario (Nombre) VALUES (@nombre)"
   OleDbParameter param = new OleDbParameter("@nombre", txtNombre)
   RESULTADO: Solo se inserta el texto como dato, no como código

DIFERENCIA:
Con concatenación ? El SQL se construye con el valor
Con parámetros    ? El SQL está completo, el valor se pasa por separado
```
